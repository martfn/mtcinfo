O que é Bootstrap

Framework front-end open-source

Características

  • Framework gratuito e open-source.
  • Simplifica e acelera o desenvolvimento de interfaces responsivas.
  • Baseado em HTML, CSS e JavaScript.
  • Uniformiza estilos com o Reboot.
  • Adota a filosofia Mobile First.
  • Usa uma grelha responsiva de 12 colunas.

História

  • Criado por Mark Otto e Jacob Thornton.
  • Originalmente chamado Twitter Blueprint.
  • Lançado publicamente em Agosto de 2011.
  • Versão em foco: 5.3.8.
🔗

Integração

CDN e local
Regra de ouroO CSS do Bootstrap é obrigatório; o JavaScript do Bootstrap também é obrigatório para componentes interativos como dropdowns, tabs, navbar colapsável e carousel.
CDN
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css" rel="stylesheet">
<link href="css/meusestilos.css" rel="stylesheet">
<script src="https://cdn.jsdelivr.net/npm/@popperjs/core@2.11.8/dist/umd/popper.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js"></script>
Local
<link href="css/bootstrap.css" rel="stylesheet">
<link href="css/meusestilos.css" rel="stylesheet">
<script src="js/bootstrap.js"></script>

Container

Base do layout
ClasseComportamento
.containerCentraliza o conteúdo e aplica uma largura máxima por breakpoint.
.container-fluidOcupa toda a largura disponível.
.container-lgFluid abaixo de lg; comporta-se como container em lg e acima.

Grid e colunas

Container → Row → Col

O layout pensa-se em linhas horizontais, e cada linha reparte o espaço por 12 colunas. As rows têm de estar dentro de um container.

<main class="container">
  <section class="row">
    <article class="col-6">Caixa 1</article>
    <article class="col-6">Caixa 2</article>
  </section>
</main>
ClasseLeitura correta
colDivide o espaço igualmente entre as caixas da row.
col-6Ocupa 6 colunas em todos os ecrãs.
col-sm-8 col-lg-6 col-xl-312 colunas em xs, 8 em sm/md, 6 em lg, 3 em xl.
col-sm-autoA largura adapta-se ao conteúdo a partir de sm.
↔

Alinhamento

Horizontal e vertical

Horizontal na row

  • justify-content-start
  • justify-content-center
  • justify-content-end
  • justify-content-between
  • justify-content-around
  • justify-content-evenly

Vertical

  • align-items-start, center, end, stretch.
  • align-self-start, center, end, stretch.
🪆

Nesting

Grelhas dentro de grelhas

Uma row dentro de uma col cria uma nova grelha com 12 colunas. É útil para subdividir uma área do layout sem sair da estrutura Bootstrap.

<section class="row">
  <article class="col-sm-6">Caixa normal</article>
  <article class="col-sm-6">
    <section class="row">
      <article class="col-sm-3">A</article>
      <article class="col-sm-9">B</article>
    </section>
  </article>
</section>

Texto e links

.text-danger, .text-opacity-75, .link-primary.

Fundo

Tipografia

Peso, estilo, alinhamento e tamanho

Peso e estilo

  • fw-bold, fw-bolder, fw-normal, fw-light, fw-lighter.
  • fst-italic, fst-normal.

Tamanho

  • fs-1 a fs-6.
  • display-1 a display-4.

Alinhamento e transformação

text-start, text-center, text-end, text-uppercase, text-lowercase, text-capitalize.

Borders

Aditivo e subtrativo

border adiciona contorno em todos os lados. border-top e border-bottom-0 permitem construir e remover lados específicos. border-1 a border-5 controlam a espessura; rounded e rounded-pill controlam o raio.

Navbar

Barra de navegação
<nav class="navbar navbar-expand-md bg-dark navbar-dark">
  <div class="container">
    <a class="navbar-brand" href="index.html">Marca</a>
    <ul class="navbar-nav">
      <li class="nav-item"><a class="nav-link active" href="#">Produtos</a></li>
      <li class="nav-item"><a class="nav-link disabled" href="#">Contactos</a></li>
    </ul>
  </div>
</nav>
Ponto-chave.navbar-expand-md faz a navbar colapsar abaixo de md. O botão toggler usa data-bs-toggle="collapse" e data-bs-target.
📑

Nav Tabs

Navegação por tabs

As tabs podem ser só visuais ou podem controlar painéis com JavaScript.

<ul class="nav nav-tabs">
  <li class="nav-item"><a class="nav-link active" href="#">Active</a></li>
</ul>

<button class="nav-link active" data-bs-toggle="tab" data-bs-target="#home-tab-pane">Home</button>

<div class="tab-content">
  <div class="tab-pane fade show active" id="home-tab-pane">...</div>
</div>
📝

Forms

Campos e layout

Estrutura base

form-label, form-control e form-text são as classes-chave.

Extras

form-check-input, form-check-label, form-control-lg, form-control-sm, input-group.

<label for="email" class="form-label">e-mail</label>
<input type="email" class="form-control" id="email" name="email">
<div class="form-text">Não partilhamos o seu e-mail</div>
🔘

Botões e badges

Componentes muito frequentes

As classes de botão podem ser usadas em button, a e input. Existem versões btn-outline-*, btn-lg, btn-sm, estados active e disabled.

<button type="button" class="btn btn-primary">Primário</button>
<button type="button" class="btn btn-outline-success">Outline</button>
<button type="button" class="btn btn-danger btn-sm">Pequeno</button>
<span class="badge bg-warning text-dark">Novo</span>
<span class="badge rounded-pill bg-success">Ativo</span>
🖼

Carousel

Imagens, indicadores e controlos

Os slides precisam de imagens com dimensões idênticas para evitar que o layout salte. O carousel pode ter indicadores, controlos e caption.

<div id="demo" class="carousel slide carousel-fade" data-bs-ride="carousel">
  <div class="carousel-indicators">...</div>
  <div class="carousel-inner">
    <div class="carousel-item active">
      <img src="carousel1.jpg" class="d-block w-100">
      <div class="carousel-caption">...</div>
    </div>
  </div>
</div>
⚠

Alerts

Mensagens para o utilizador
<div class="alert alert-warning alert-dismissible fade show">
  <h4 class="alert-heading">Alerta!</h4>
  <p>Este site usa cookies...</p>
  <button type="button" class="btn-close" data-bs-dismiss="alert"></button>
</div>

O texto do alerta pode ser fechado com um botão ou com um link com data-bs-dismiss="alert".

Responsive

img-fluid faz a imagem adaptar-se à largura do contentor sem ultrapassar as suas dimensões originais.

Efeitos

rounded, rounded-circle, rounded-pill e img-thumbnail.

Como responder no teste

Teórico e prático

Teste teórico

  • Identificar características do Bootstrap.
  • Explicar diferenças entre container, container-fluid e breakpoints.
  • Reconhecer classes de grid, cores, tipografia e componentes.

Teste prático

  • Prever o que o browser vai mostrar.
  • Apontar problemas no código.
  • Corrigir classes e estrutura.
  • Propor melhorias funcionais e responsivas.
Checklist rápidoVerifica sempre: ordem container → row → col, uso correto dos breakpoints, classes obrigatórias dos componentes e dependência de JavaScript quando o componente for interativo.
