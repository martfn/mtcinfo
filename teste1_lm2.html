Teste teórico – T01 a T06

T01 – Contexto Web & camadas

Arquitetura cliente/servidor, protocolos Web, browsers e standards W3C. Base para o resto da matéria.

O browser é o cliente: faz pedidos HTTP ao servidor. O servidor responde com HTML, CSS e JS, que o browser interpreta e renderiza.

HTML = estrutura. CSS = apresentação. JS = comportamento. Três camadas separadas, mas que trabalham em conjunto.

RWD parte do desktop e adapta para mobile. Mobile-first parte do mobile e expande para ecrãs maiores.

O W3C define standards que os browsers seguem para garantir compatibilidade, acessibilidade e boa experiência de utilização.

T02/T03 – Estrutura HTML5 & CSS base

Estrutura base de uma página HTML5, tags semânticas, imagens, hiperligações, seletores básicos, cores e tipografia CSS.

<!DOCTYPE html>
<html lang="pt">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Título</title>
    <link rel="stylesheet" href="estilos.css">
  </head>
  <body>
    <header>Cabeçalho</header>
    <nav>Navegação</nav>
    <main>
      <section>
        <article>Conteúdo</article>
      </section>
    </main>
    <aside>Conteúdo lateral</aside>
    <footer>Rodapé</footer>
  </body>
</html>
  • <header> – cabeçalho da página/section.
  • <nav> – navegação principal.
  • <main> – conteúdo principal (único).
  • <section> – grupo temático.
  • <article> – conteúdo autónomo.
  • <aside> – conteúdo lateral.
  • <footer> – rodapé.
<!-- CSS externo, interno, inline -->
<link rel="stylesheet" href="estilos.css">
<style> p { color: blue; } </style>
<p style="color:red">Texto</p>

Prioridade (sem !important): inline > interna > externa.

T03/T04 – Box Model, unidades e seletores

Complementa T02 com formatação de imagens, pseudo-classes de links, display block/inline, tabelas, listas e unidades absolutas/relativas.

a {
  text-decoration: none;
  font-size: 14px;
}
a:link    { color: red; }   /* não visitado */
a:visited { color: green; } /* já visitado */
a:hover   { color: hotpink; } /* rato por cima */
a:active  { color: blue; }  /* ao clicar */

A ordem LoVe HAte (:link, :visited, :hover, :active) é obrigatória para o CSS funcionar corretamente.

Tipo Quebra linha? Controla width/height? Exemplos
block Sim Sim div, p, h1, section
inline Não Não span, a, strong
inline-block Não Sim img (por defeito)
Unidade Tipo Relativo a Uso típico
px Absoluta Pixel Linhas, bordas
em Relativa font-size do pai Padding/margens relativas ao texto
rem Relativa font-size do html Escala global
% Relativa Elemento pai Larguras fluidas
vh/vw Relativa Viewport Alturas e larguras fullscreen

T06 – Overflow, sprites, transformações

Controlar overflow, usar sprites para ícones e aplicar transformações 2D/3D.

section {
  width: 200px;
  height: 200px;
  overflow: scroll; /* ou auto, hidden */
}
.icon {
  width: 30px;
  height: 30px;
  background-image: url("sprite.png");
  background-position: -44px -180px;
}

Transições e animações são usadas para micro-interações (hover em links, inputs com :focus, etc.).

Teste prático – Ex. 4 & 5

Formato do teste prático
  • Em papel, 60 minutos, após o teórico.
  • Perguntas de desenvolvimento sobre fragmentos de código HTML/CSS.
  • Tarefas: detectar erros, explicar seletores, prever aparência, propor correções.
O enunciado é reduzido; o importante é mostrar que compreendes o que o código faz.
Ex. 4 – Mark Ryden (4a/4b)
Ex. 4 – páginas a rever: index.html (4a) + participate.html (4b).
  • 4a – index.html: site sobre Mark Ryden com estrutura semântica (header/nav/main/section/article/figure/footer).
  • Layout centrado com width: 1000px e margin: auto.
  • Seletores avançados (descendente, child, pseudo-classes, attribute) e heading h2 com hover que inverte cores sem provocar salto.
  • 4b – participate.html: página com formulário completo que envia via method="post" para https://formtester.goodbytes.be/post.php.
  • Campos: First name, Last name, Email, Educational level (select), Age (number), radio (Drawings/Paintings), 6 checkboxes, textarea e botões submit/reset.
Ex. 5 – Blog Mr. Fig (5a/5b)
Ex. 5 – páginas a rever: index.html + article1.html (5a), about.html + 404.html (5b).
  • 5a – index.html + article1.html: blog com aside e header fixos, main deslocado com margens.
  • Combinação de position: fixed, relative e absolute, e uso de z-index em elementos sobrepostos.
  • Estilos no nav (links com border-bottom e transição) e input de pesquisa com transição em largura e cor no :focus.
  • 5b – about.html: section com height: 75vh, overflow-y: scroll e borda tracejada.
  • 5b – 404.html: animação em dino.gif com @keyframes, duração 3s, forwards, e translação no eixo X.
Tipos de pergunta no prático
  • "O que aparece no browser?" – descrever cores, posições, tamanhos, estados de links, etc.
  • "O que está errado?" – apontar problemas: falta de name, seletor errado, ordem de pseudo-classes, Box Model mal calculado.
  • "O que esta regra faz?" – explicar seletor/propriedade (posição, overflow, transition, animation).
  • "Propõe uma correção" – escrever o código correto para o comportamento descrito.

CSS essencial

Formas de aplicar CSS
<!-- Externo (recomendado) -->
<link rel="stylesheet" href="estilos.css">

<!-- Interno / embedded -->
<style> p { color: blue; } </style>

<!-- Inline (evitar) -->
<p style="color: red;">Texto</p>

Prioridade (sem !important): inline > interno > externo. Especificidade: id > classe/pseudo-classe > tag.

Box model & centralização
.box {
  width: 200px;
  padding: 20px;
  border: 5px solid red;
  margin: 10px auto; /* centra horizontalmente */
}

Para centrar um elemento block: precisa de largura definida e margin-left/right: auto.

Seletores importantes
article p   { } /* descendente */
article > p { } /* filho direto */
article + p { } /* irmão adjacente */
article ~ p { } /* irmãos gerais */

a[href^="https://"] { } /* começa com */
a[href$=".pdf"]    { } /* termina em */
a[href*="aveiro"]  { } /* contém */

p:first-child    { }

p:nth-child(odd) { }

Overflow, transição e animação
section {
  height: 75vh;
  overflow-y: scroll;
}

  border-bottom: 5px solid white;
  transition: border-bottom 1s ease;
}

}

  0%   { transform: translateX(0); }
  100% { transform: translateX(150%); }
}

  width: 400px;
